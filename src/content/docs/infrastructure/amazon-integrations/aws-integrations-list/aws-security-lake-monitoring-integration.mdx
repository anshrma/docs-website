[New Relic infrastructure integrations](/docs/infrastructure/integrations-getting-started/getting-started/introduction-infrastructure-integrations) 
now include an integration for AWS Security Lake, allowing you to send your security log data to New Relic. 

Collect and send telemetry data to New Relic from [Security Lake](https://aws.amazon.com/security-lake/) using our integration. You can use this integration to monitor your services, query incoming data, and build dashboards to observe everything at a glance.

## 1. Activate integration [#activate]

1. Complete the steps mentioned in Amazon Security Lake [Getting Started Guide]( https://docs.aws.amazon.com/security-lake/latest/userguide/getting-started.html).
2. Complete the pre-requsites mentioned under `data access` subscriber type as mentioned at [Security Lake Subscriber Guide](https://docs.aws.amazon.com/security-lake/latest/userguide/subscriber-data-access.html#prereqs-creating-subscriber)
3. Consolidate data from multiple regions into a single rollup. We recommend consolidating multiple regions so you can avoid repeating the steps in the next section for each region. 

Amazon Security Lake collects security logs and events across AWS Regions for enabled services. AWS stores data in different Amazon S3 buckets per each region. For more details, see [managing multiple regions](https://docs.aws.amazon.com/security-lake/latest/userguide/manage-regions.html).

## 2. Create a subscriber with data access [#subscriber]

Follow the below steps to give New Relic data access to your rollup: 

  1. Open the [Security Lake console](https://console.aws.amazon.com/securitylake/).
  2. Select the region where you rolled up the Security Lake logs by clicking the AWS Region selector in the upper-right corner of the page.
  3. Choose **Subscribers** in the navigation pane.
  4. Choose `Create subscriber` on the **Add subscribers** tab,
  5. Mention `new-relic` as the subscriber name under subscriber details. Add `new-relic` in the  **(optional)** description as well.
  6. For Log and event sources, select `All log and event sources`. This fowards logs to New Relic from all your configured sources. 
  7.  Choose `S3` for Data Access method. This gives data access to the subscriber, in this case `new-relic`. 
  8. For Subscriber credentials:
    * Input the AWS account ID provided by New Relic so you can forward the logs aggregated by Security Lake. This is likely where you installed the AWS Lambda function. 
    * [External ID](https://docs.aws.amazon.com/security-lake/latest/userguide/subscriber-data-access.html#subscriber-external-id).
  9. Select `SQS queue` for notification details.
  10. Choose **Create**.

Note the `AWS Role ID`, `External ID`, `Account ID` and `Subscription endpoint`

## 3. Set up the log forwarder [#setuplog]

From the AWS account that has access to the regions rollup:

1. Install our dedicated serverless application [Amazon Security Lake log forwarder](/docs/logs/forward-logs/aws-lambda-sending-security-logs-s3/). 
2. Open the [AWS Serverless Application Repository](https://serverlessrepo.aws.amazon.com/applications) in your browser.
3. Search for `newrelic` and check **Show apps that create custom IAM roles or resource policies** to find `newrelic-securitylake-s3-processor-LogForwarder`.
4. Click the `newrelic-securitylake-s3-processor-LogForwarder` details and click **Deploy**. Ensure you are in the AWS region where Security lake logs are getting rolled up.
5. Provide the following parameters to set the forwarder lambda function.
  * SecurityLakeSQSArn : `Subscription endpoint` from step 4 (as shown in the example)
  * SecurityLakeSubscriberRoleArn : `AWS Role ID` from step 4 (as shown in the example)
  * ExternalID : `External ID` from step 4 (as shown in the example)
  * NRLicenseKey : <InlinePopover type="licenseKey" />

## Logs [#logs]

To find your logs on New Relic, go to **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Logs** and set **Attributes** to `product.name`, choose one or few of your interest.

The followings source types of logs are currently supported:

* [ROUTE 53](https://docs.aws.amazon.com/Route53/latest/DeveloperGuide/logging-monitoring.html)
* [VPC FLOW](https://docs.aws.amazon.com/vpc/latest/userguide/flow-logs.html)
* [CLOUDTRAIL](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/cloudtrail-getting-started.html)
* [SECURITY HUB](https://docs.aws.amazon.com/securityhub/latest/userguide/what-is-securityhub.html)


<Callout variant="info">
  Amazon Security Lake uses the [OCSF Schema](https://schema.ocsf.io/) for it's logs.
</Callout>
